# Kwork Lead Funnel

Локальное приложение для быстрого поиска подходящих Kwork-заказов на web-разработку. Оно проверяет свежесть и число предложений, читает доступное ТЗ и вложения, готовит человеческий черновик отклика через DeepSeek и отправляет карточку в мобильное приложение Никиты.

## Рабочий поток

1. Фоновый исполнитель на ПК держит отдельную Kwork Chrome-сессию и принимает команды из раздела `Заказы` в family_todo: `Сканировать сейчас`, `Старт`, `Стоп`.
2. Сканер читает свежие заказы, не создаёт дублей, отбрасывает Bitrix и карточки с числом предложений выше `KWORK_MAX_RESPONSES`.
3. Для подходящего заказа он открывает саму страницу Kwork, читает видимый текст, ТЗ, документы, архивы и скриншоты, затем готовит оценку и черновик.
4. Новый лид сохраняется в SQLite и публикуется в защищённый API мобильного приложения для номера `+7 967 981 2438`.
5. На телефоне в разделе `Заказы` можно открыть исходный заказ, проверить описание и вложения, поправить текст, название, цену и срок.
6. Кнопка `Одобрить и отправить с ПК` сначала сохраняет текущий текст, название, цену и срок, затем передаёт единственную команду на ПК. Он повторно проверяет Kwork, заполняет форму и отправляет отклик. Статус `Отправлен` или причина ошибки возвращаются в мобильную карточку.

Почта в рабочем маршруте не используется. Без мобильного одобрения внешний отклик не отправляется.

## Запуск

Основной способ: запусти ярлык `lead-funnel-gui.vbs`. Он открывает окно без терминала. Для управления с телефона исполнитель `start_mobile_control.cmd` добавлен в автозапуск текущего пользователя и не сканирует Kwork, пока не получит команду из family_todo.

- `Открыть Kwork Chrome` - запускает отдельный профиль Chrome для Kwork. Войди в Kwork один раз именно в этом окне.
- `Сканировать` - один проход по текущим свежим заказам.
- `Старт мониторинга` - повторяет проверку с интервалом из настроек и забирает одобренные с телефона команды.
- `Стоп` - останавливает мониторинг.
- Вкладка `Лиды` остаётся локальным редактором и журналом Kwork-откликов.

Запасной запуск из терминала:

```powershell
.\lead-funnel.cmd scan
.\lead-funnel.cmd watch
```

## Настройка

1. Скопируй `.env.example` в `.env`.
2. Заполни `DEEPSEEK_API_KEY`.
3. Оставь параметры mobile inbox:

```dotenv
LEAD_HUB_URL=http://31.129.97.211
LEAD_HUB_API_KEY=dev-local-key
LEAD_HUB_OWNER_PHONE=79679812438
LEAD_HUB_EXECUTOR_ID=kwork-desktop
```

4. Настрой отбор в GUI или в `.env`:

```dotenv
SCAN_INTERVAL_SECONDS=60
KWORK_MAX_RESPONSES=5
KWORK_MAX_AGE_HOURS=24
LEAD_MAX_DAYS=7
LEAD_BLOCKED_KEYWORDS=битрикс,bitrix
```

`KWORK_MAX_RESPONSES` проверяется при поиске и прямо перед отправкой. Если Kwork не показывает число предложений, заказ не получает автоматический отклик.

Для приватных страниц и файлов используется отдельная Kwork Chrome-сессия. `KWORK_LOGIN_EMAIL` и `KWORK_LOGIN_PASSWORD` нужны только как fallback, если сохранённая сессия вышла; при captcha или SMS потребуется ручное подтверждение в открытом окне.

## Вложения

Сканер пытается разобрать `pdf`, `docx`, текстовые файлы, `zip`, `rar`, `7z` и изображения. Для изображений используется Tesseract на диске `D:`:

```dotenv
TESSERACT_CMD=D:\Tesseract-OCR\tesseract.exe
```

OpenRouter дополняет OCR и сложные скриншоты, когда в `.env` заполнен `OPENROUTER_API_KEY`. В карточке видно, что именно было скачано, прочитано, раскрыто из архива или распознано OCR.

## Дедупликация и статусы

Локальная SQLite-база не создаёт второй лид для того же Kwork project ID после перезапуска. На сервере каждая мобильная карточка имеет свой `external_key`, аудит изменений и статус: `new`, `edited`, `approved`, `sending`, `sent`, `rejected` или `failed`.

Одобренную карточку может захватить только один ПК-исполнитель. Повторное нажатие `Одобрить` и повторный проход мониторинга не создают второй Kwork-отклик.
